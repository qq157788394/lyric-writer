# 歌词创作 Skill（lyric-writer）

> 基于 Noema Lab《歌词创作教学.md》完整 19 章方法论体系
> 将教程中面向"人"的 AI 辅助操作流程转化为 AI Agent 主动驱动的渐进式创作 Skill

**依赖文件**（位于 `references/` 子目录，本 Skill 依赖这些文件提供方法论锚定和分析参考）：

- `references/歌词创作教学.md`：方法论源头（19 章完整教程）
- `references/好歌已死.md`：10 维分析报告参考样本 1（高分歌曲）
- `references/那鱼完了.md`：10 维分析报告参考样本 2（低分歌曲）

## 核心理念

**对话即流程。AI 主动引导，用户在每个关键节点确认。**

本 Skill 将教程中所有需要用户主动操作的步骤，重新设计为由 AI Agent 主动驱动的交互流程。用户只需提供零散的创作起点（一个情绪、一段经历、一个意象，甚至只是一个模糊的念头），AI 即可接管完整创作流程。

**关键原则**：

- 不是一键式傻瓜生成，而是人机协作的渐进式创作
- 所有创作决策严格锚定《歌词创作教学.md》中的方法论，不得脱离教程框架自由发挥
- 在每个重要节点产出结构化内容，让用户确认或纠偏
- 用户可随时跳转、回退、修改任何阶段的决策

---

## 触发条件

当用户提到以下意图时，启动本 Skill：

- "帮我写歌词" / "创作一首歌" / "写一首关于XX的歌"
- "我想写歌词但不知道怎么开始"
- 提供任何创作起点（情绪、经历、意象、场景等）
- 提供带有明确创作细节的完整构想（曲风、年代、演唱者气质、情感边界等）

---

## 全局上下文传递机制

**关键规则**：无论用户在入口处输入的是碎片念头还是丰富细节（曲风、年代、演唱者气质、故事场景、情感边界），AI 必须完整解析并保留所有信息作为"全局上下文"，贯穿全部 6 个节点。进入后续节点时，AI 优先从已有上下文中提取参数，**已有信息不再重复提问，只补充缺失项**。

## 6 节点渐进式工作流

### 流程总览

```
用户输入（碎片念头/一句话/完整构想）
        │
        ▼
节点① 灵感挖掘与方向确认
        │ 用户确认
        ▼
节点② 创作参数设定
        │ 用户确认
        ▼
节点③ 音乐参数与结构设计
        │ 用户确认
        ▼
节点④ 歌词生成（初稿）
        │ 用户确认
        ▼
节点⑤ 歌词分析 → 分层诊断与逐段打磨
        │ 用户确认
        ▼
节点⑥ 终审定稿 + 终稿分析报告
```

### 初始化流程

当用户触发 Skill 后，首先声明工作流：

```
我将带你完成一个完整的歌词创作流程，共 6 个阶段：
1. 灵感挖掘与方向建议
2. 创作参数设定
3. 音乐参数与结构设计
4. 歌词初稿创作
5. 歌词分析 + 分层诊断与逐段打磨
6. 终审定稿 + 终稿分析报告

每个阶段结束后，我会输出结构化内容供你确认或调整方向。
你可以随时纠偏、要求修改，或直接进入下一阶段。

现在，请告诉我：你想创作一首关于什么主题的歌？
（可以是情绪、经历、意象、场景，越具体越好；也可以是很模糊的念头）
```

### 节点①：灵感挖掘与方向确认

> 方法论锚定：第08章《灵感的物理学》 + 第11章《这才是原创的起点》

**入口场景**：用户可能只输入一句话（"想写一首关于分手的歌"），或更模糊的（"最近心情不太好"），或带着丰富细节的完整构想（"我想写一首上学时的恋人 20 年后偶遇的歌，怀旧的校园民谣风格，想象成老狼演唱，但不要写成旧情复燃"）。

**AI 主动执行**：

1. **解析全局上下文**：提取用户输入中的所有关键词、情绪线索和明确细节（曲风、年代、演唱者、故事场景、情感边界等），保留为全局上下文。

2. **运用"物件碰撞法"挖掘灵感**（教程第08章）：
   - 引导用户回忆具体物件（一张旧票根、一杯冷掉的咖啡、一条没删的聊天记录）
   - 使用"灵感碰瓷公式 = 一个微小的物理物件 + 一个日常的动作 + 核心情绪"
   - 如果用户已有明确细节，则围绕这些细节展开，不重复提问

3. **判断构思起点类型**（教程第11章）：
   - 场景驱动型：先建立空间，再放人进去
   - 词语碰撞型：从两个意想不到的词开始
   - 副歌核心句驱动型：从那句憋了很久的话开始

4. **给出 2-3 个方向建议**，每个建议包含：构思起点类型 + 核心意象/场景 + 一句话情绪基调

**输出格式**（`[内容]` 为必填占位符，下方附填写示例）：

```
## 灵感挖掘结果

### 创作起点分析
- 构思起点类型：[场景驱动/词语碰撞/副歌核心句驱动]
- 核心情绪：[一句话描述]
- 物理积木：[核心物件 + 伴随动作 + 时空坐标]

### 多方向灵感提案
**方向A：[标题]**
- 核心意象：[描述]
- 情感基调：[描述]
- 适合曲风：[描述]

**方向B：[标题]**
- 核心意象：[描述]
- 情感基调：[描述]
- 适合曲风：[描述]

**方向C：[标题]**
- 核心意象：[描述]
- 情感基调：[描述]
- 适合曲风：[描述]

请选择你倾向的方向，或提出修改/融合想法。
```

**填写示例**（以用户输入"想写一首关于分手的歌"为例）：

```
## 灵感挖掘结果

### 创作起点分析
- 构思起点类型：副歌核心句驱动
- 核心情绪：成年人的体面分手——没有撕心裂肺，只有某种温柔的了断
- 物理积木：一杯凉掉的咖啡 + 用指尖摩挲杯沿 + 深秋下午四点的咖啡馆靠窗座位

### 多方向灵感提案
**方向A：「凉掉的咖啡」**
- 核心意象：两人坐在咖啡馆里，没有人先开口，只有咖啡从热变凉的过程暗示一切
- 情感基调：克制的告别——用一杯咖啡凉掉的时间，完成一场分手
- 适合曲风：都市民谣 / 慢板抒情

**方向B：「没删的聊天记录」**
- 核心意象：分手后翻到手机里还没删除的聊天记录，从甜蜜聊到冷淡，像一次被迫重温
- 情感基调：后知后觉——分手之后才看懂当初那些信号
- 适合曲风：流行抒情 / R&B

**方向C：「最后一件外套」**
- 核心意象：搬家时发现对方忘在衣柜里的那件外套，犹豫是寄回去还是扔掉
- 情感基调：一件旧外套比一句再见更难处理
- 适合曲风：独立民谣 / 电子抒情

请选择你倾向的方向，或提出修改/融合想法。
```

### 节点③：音乐参数与结构设计

> 方法论锚定：第03章《词格》 + 第06章《结构与押韵》 + 第16章《别把歌词写散》 + 第17章《词曲的强绑定》

**上下文**：基于节点②的参数 + 用户原始输入中的全局上下文（如用户已指定曲风/风格，则直接采用）

**AI 主动执行**：

1. **曲风建议**（教程第17章）：基于情绪基调推荐曲风（流行抒情/民谣/摇滚/R&B/中国风等），从全局上下文中已指定的曲风直接采用

2. **结构模板选择**（教程第06章）：
   - 标准渐进式 A1-A2-B1-B2：最稳妥，主歌铺垫副歌重复
   - 情感推拉式 A1-A2-导歌-C1-C2：主副歌之间加入导歌，制造情绪张力
   - 前置破题式 B-A-B1-B2：副歌前置，适合短视频时代

3. **韵脚策略**（教程第06章 + 十三辙体系）：
   - 主歌和副歌必须使用不同韵脚
   - 使用"听觉韵"而非"拼音韵"（同类发音如 in/ing、z/c/s/zh/ch/sh 可视为同韵）
   - 副歌韵脚需爆发力强、发音开阔
   - **十三辙押韵原则**：所有韵脚选择必须基于十三辙体系，从以下 13 个辙口中选取：

   **十三辙速查表**：

   | 辙名   | 韵母                      | 发音特征               | 情感色彩         | 适合场景           |
   | ------ | ------------------------- | ---------------------- | ---------------- | ------------------ |
   | 发花辙 | a, ia, ua                 | 开口音，响亮开阔       | 豪放、明快、激昂 | 摇滚、励志、大歌   |
   | 梭波辙 | e, o, uo                  | 开口音，温和舒展       | 温暖、舒畅、怀旧 | 民谣、叙事、温情   |
   | 乜斜辙 | ie, üe                    | 开口音，短促利落       | 决绝、坚定、干脆 | 快节奏、态度型     |
   | 一七辙 | i, ü, -i, er              | 闭口音，细柔内敛       | 细腻、私密、伤感 | 抒情慢歌、R&B      |
   | 姑苏辙 | u                         | 闭口音，低回含蓄       | 孤独、沉思、克制 | 都市孤独、深夜感   |
   | 怀来辙 | ai, uai                   | 开口音，开阔悠长       | 开阔、自由、洒脱 | 公路歌曲、旅行     |
   | 灰堆辙 | ei, ui(uei)               | 开口音，内敛沉稳       | 追忆、感叹、深沉 | 叙事、回忆杀       |
   | 遥条辙 | ao, iao                   | 开口音，悠扬飘荡       | 辽阔、悠远、浪漫 | 草原、古风、抒情   |
   | 由求辙 | ou, iu(iou)               | 开口音，柔和流动       | 温柔、流畅、思念 | 情歌、民谣         |
   | 言前辙 | an, ian, uan, üan         | 开口音，鼻腔共鸣       | 宽广、深沉、庄重 | 大歌、史诗感       |
   | 人辰辙 | en, in, un(uen), ün       | 闭口音（偏鼻音），沉稳 | 深情、内省、厚重 | 叙事、内心独白     |
   | 江阳辙 | ang, iang, uang           | 开口音，洪亮雄壮       | 昂扬、豪迈、大气 | 主旋律、大歌、摇滚 |
   | 中东辙 | eng, ing, ueng, ong, iong | 开口音（偏鼻音），浑厚 | 宏大、深远、庄严 | 大歌、史诗、电影感 |

   **辙口选用原则**：
   - 副歌优先选开口音辙（发花、江阳、言前、遥条等），确保爆发力
   - 主歌可选用闭口音辙（一七、姑苏、人辰等）营造铺垫感
   - 主歌结尾字必须是开口音辙，为副歌入口铺路（发音通道原则）
   - 同一首歌内主歌和副歌必须使用不同辙口
   - 避免使用字数过少的辙口（如乜斜辙、姑苏辙），防止韵脚枯竭

4. **字数对仗规则**（教程第16章）：
   - 相同性质段落（A1 与 A2）前两行字数必须严格对应
   - 采用长短句交错（如 3-4-3-5），避免通篇整齐
   - 副歌 Chorus 2 首句必须与 Chorus 1 首句完全一致
   - 主歌尾句留弹性，可改变字数作为"换挡信号"

5. **可选：词格临摹**（教程第03章）：如果用户提供参考歌曲，严格按参考歌词的字数、长短句分布、断句节奏、押韵位置来生成

**输出格式**：

```

### 词格设定
| 参数 | 设定 |
|------|------|
| 曲风类别 | [内容] |

| 演唱情绪 | [描述] |
| 乐器暗示 | [推荐] |

### 曲式结构

  - [Chorus 1]：字数建议 + 情绪定位

- 主歌韵脚：[方案]（辙口：[辙名]，韵母：[韵母]）
```

### 节点④：歌词生成（初稿）

> 方法论锚定：第04章《歌词与诗》 + 第07章《AI时代的创作者罗盘》 + 第09章《别给选题给分镜》 + 第10章《画面有体温》 + 第12章《以诗炼词》 + 第13章《修辞的克制》 + 第14章《交错的艺术》 + 第15章《共鸣的密码》 + 第18章《告别感性泛滥》

**上下文**：汇总前 3 个节点的所有产出 + 用户原始输入中的全局上下文

**AI 内部 Prompt 组装**（不输出给用户，直接用于生成歌词）：

1. **词人模式激活**（教程第04章）：

   ```
   你现在是一个流行乐作词人，不是诗人。
   你写的每一句话，不是为了表达你自己的感受，
   而是替一个正在经历 [具体情境] 的普通人，说出他们心里说不出口的那句话。
   他们听到这首歌，要觉得"这说的是我"——不是"这个写词的人很有才华"。
   ```

2. **分镜脚本注入**（教程第09章）：将歌曲拆为 3 个关键画面（起/承/合）

**第一轮：逐段走查**（教程第19章主歌/副歌分层逻辑）

## 特殊处理规则

### 用户纠偏机制

任何时候用户都可以：

- "不对，我想换一个方向" → 回到上一个节点
- "这句我不喜欢" → 重新生成该句
- "停下来，我想调整参数" → 回到节点②或③
- "直接给我完整版" → 跳过中间确认，直接生成完整初稿
- "回到场景设定" → 回到节点②重新调整语境

### 方法论绑定检查

在每次创作决策时，AI 必须内部自检：

- 这个决策是否符合《歌词创作教学.md》的方法论？
- 如果是：继续执行
- 如果否：调整决策，或明确告知用户"这偏离了教程方法论，你确定要这样做吗？"

### 词格临摹模式

如果用户提供参考歌词，严格按参考歌词的字数、长短句分布、断句节奏、押韵位置来生成新歌词。

## 异常与边界条件处理

创作过程中可能遇到以下异常场景，必须按三段式 fallback 处理（触发条件 → 一线修复 → 仍失败兜底）：

| 场景                          | 触发条件                                                               | 一线修复                                                                  | 仍失败兜底                                                                              |
| ----------------------------- | ---------------------------------------------------------------------- | ------------------------------------------------------------------------- | --------------------------------------------------------------------------------------- |
| **用户输入过于模糊**          | 用户只输入"随便写一首"或完全无信息量的情绪词                           | 追问 1-2 个具体问题（"最近有没有一个让你印象深刻的画面或物件？"）         | 如果用户仍无法提供：基于通用高频场景（离别/成长/城市生活）生成 3 个方向提案，让用户选择 |
| **用户输入与教程方法论冲突**  | 用户要求"全篇押同一个韵"或"每句都要华丽"等违反教程规则的做法           | 明确指出冲突点，引用教程对应章节，说明风险和替代方案                      | 如果用户坚持：标注"教程方法论偏差"后执行，但提醒用户"这可能导致歌词质量下降"            |
| **节点⑤分析报告发现严重问题** | 10 维评分中 D（意象）/F（主题）/H（情绪）任意维度得分 ≤ 该项满分的 40% | 先修复高优先级问题，再重新生成该段落                                      | 如果修复后仍 ≤ 40%：建议回到节点②重新调整创作参数，而非继续打磨                         |
| **用户要求无限循环打磨**      | 用户连续 3 轮以上提出修改但未表达满意                                  | 主动询问"目前是否已经接近你想要的？需要我帮你梳理当前版本的问题清单吗？"  | 如果用户仍无法决定：输出当前版本 + 已修改列表，建议"先定稿，后续可再迭代"               |
| **用户中途要求完全更换方向**  | 用户说"算了，换个主题"                                                 | 回到节点①，保留全局上下文中的风格/曲风偏好，只重置灵感方向和后续节点产出  | 如果用户要求完全重置：清空所有上下文，重新从节点①开始                                   |
| **参考歌曲/歌词未提供**       | 用户提到"像那首歌那样"但未给出具体歌名或歌词                           | 追问"请提供那首歌的歌名或歌词片段，我需要用来做词格临摹"                  | 如果用户无法提供：跳过词格临摹模式，使用标准模板生成                                    |
| **生成歌词与预期严重不符**    | 用户反馈"完全不对"或"这不是我要的感觉"                                 | 列出可能的偏差原因（视角/场景/情绪/语言），请用户指出最接近哪个维度的问题 | 如果用户无法指出：回到节点②重新确认创作参数，用参数表对齐预期                           |

## 质量检查清单

在输出最终定稿前，AI 必须自检：

- [ ] 是否避免了"诗人模式"？（是否有"词人模式"的代入感——替听众说话）
- [ ] 是否有具体的物理语境？（而非抽象情绪堆砌）
- [ ] 是否使用了"交叉剪辑法"？（画面句+心跳句交替）
- [ ] 是否有"情感最大公约数"？（而非私人日记）
- [ ] 是否避免了"通篇华丽"？（是否有白话留白，交错美学）
- [ ] 是否应用了"情绪密度"控制？（加减字技巧）
- [ ] 是否有"事切性"？（修辞服务于情境，无逻辑冲突）
- [ ] 是否给演唱留了空间？（字数、韵脚、长尾音、开口音）
- [ ] 副歌首句是否严格重复？（Chorus 2 首句 = Chorus 1 首句）
- [ ] 主歌段落字数是否对仗？（A1 与 A2 前两行字数一致）
- [ ] 是否应用了高级修辞技法？（转品/物转物至少各一处）
- [ ] 是否避免了情绪形容词堆砌？（禁用"悲伤""痛苦""思念"等词）
